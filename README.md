## (β) Τεχνικές Απαιτήσεις (Android SDK & Versions)

Η εφαρμογή έχει αναπτυχθεί και δοκιμαστεί με τις παρακάτω προδιαγραφές:

- **Dart SDK**: `^3.9.0`
- **Flutter SDK**: Latest Stable
- **Android SDK**:
  - `minSdkVersion`: 23 (Android 6.0 Marshmallow) ή νεότερο (λόγω βιβλιοθηκών όπως camera/location).
  - `targetSdkVersion`: 34 (Android 14).
  - `compileSdkVersion`: 34.
- **Google Build Image**: Απαιτείται emulator image **με Google Play Services** (π.χ. "Pixel 6 API 33 with Play Store"). Χωρίς Play Services, η λειτουργία του Firebase Auth και Maps ενδέχεται να αποτύχει.

## (γ) Διαφοροποιήσεις από το Πρωτότυπο (Phase B)

Σε σχέση με τον αρχικό σχεδιασμό (StudySpot Identity), η υλοποίηση καλύπτει τον πυρήνα της λειτουργικότητας αλλά παρουσιάζει μερικές διαφοροποιήσεις/παραλείψεις:

1. **Αυτοματοποιημένος Εντοπισμός (GPS Geofencing)**:
   - *Σχεδιασμός*: Η εφαρμογή θα ανίχνευε αυτόματα την αποχώρηση από τον χώρο μέσω GPS.
   - *Υλοποίηση*: Η αποδέσμευση της θέσης γίνεται κυρίως χειροκίνητα ή μέσω QR Code (Scan to Leave) για λόγους απλότητας και εξοικονόμησης μπαταρίας/πόρων σε αυτή τη φάση.

2. **Smart Notifications & User Modeling**:
   - *Σχεδιασμός*: Έξυπνες ειδοποιήσεις βασισμένες σε συνήθειες (π.χ. "Συνήθως έρχεσαι τέτοια ώρα").
   - *Υλοποίηση*: Υπάρχουν βασικές ειδοποιήσεις (π.χ. προσθήκη φίλου), αλλά το πολύπλοκο μοντέλο πρόβλεψης συνηθειών δεν έχει ενσωματωθεί πλήρως.

3. **Gamification (Exams Wrapped & Advanced Avatar)**:
   - *Σχεδιασμός*: "Exams Wrapped" στο τέλος εξεταστικής και Avatars που "διαβάζουν" real-time.
   - *Υλοποίηση*: Υπάρχει σύστημα Achievements και το Status (Seated/Coming Soon), αλλά η λειτουργία "Wrapped" και τα animations των avatars είναι μελλοντικές προσθήκες.

4. **Αισθητήρες (Haptics)**:
   - *Σχεδιασμός*: Εκτεταμένη χρήση δόνησης σε κάθε interaction.
   - *Υλοποίηση*: Η χρήση haptics είναι περιορισμένη σε βασικά σημεία.

Οι βασικοί άξονες (Κρατήσεις, Social Δικτύωση, Queue System, Profile) έχουν υλοποιηθεί πλήρως όπως περιγράφονται.
